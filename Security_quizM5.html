Real-World Example: United Nations Website Defacement

One example of the importance of cybersecurity involved the official United Nations website. Attackers successfully compromised part of the website and replaced official content with unauthorized political messages.

The message appeared within the section reserved for statements made by Secretary-General Ban Ki-moon. Although the website was restored shortly afterward, the incident demonstrated how even well-known organizations can become victims of cyberattacks.

Cybersecurity Term:

Hacktivism is the use of hacking techniques to promote political or social causes. While motivations vary, unauthorized access to computer systems remains illegal.

This event highlights why organizations must implement strong cybersecurity practices such as:

  • Strong authentication
  • Regular software updates
  • Encryption of sensitive data
  • Continuous monitoring
  • Employee security awareness training

Case Study: Ransomware Attack

Ransomware is malicious software that encrypts files or locks computer systems until a ransom is paid. It is one of the most damaging forms of cybercrime because it can completely disrupt business operations.

During my internship at Pridgeon & Clay, I learned about a ransomware incident that had occurred the year before I joined the company. The attackers reportedly gained access after successfully brute-forcing a server password.

Once inside the network, the attackers encrypted important company files, preventing employees from accessing critical business information. Instead of paying the ransom, the company rebuilt affected systems from backups and strengthened its overall security posture.

Lessons Learned
  • Use strong passwords.
  • Enable Multi-Factor Authentication (MFA).
  • Maintain secure offline backups.
  • Keep systems fully updated.
  • Educate employees about phishing attacks.

Cybersecurity is not only about technology—it also depends on user awareness and following good security practices every day.

Common Malware Types

Cybercriminals use many different forms of malware to compromise computers, steal information, or disrupt services. Understanding these threats is the first step toward defending against them.

Threat Description
Ransomware Encrypts files or systems until a ransom is demanded.
Trojan Horse Disguises itself as legitimate software to trick users into installing malware.
Virus Infects programs and spreads when infected files are executed.
Worm Self-replicates across networks without user interaction.
Spyware Secretly monitors user activity and collects sensitive information.
Keylogger Records every keystroke to steal passwords and personal information.
Rootkit Provides attackers with hidden administrative access to a system.
Logic Bomb Malicious code that activates only when specific conditions are met.
Bloatware Unwanted software that consumes resources and may introduce security risks.
Remember:

The best protection against malware is keeping software updated, avoiding suspicious downloads, using antivirus software, and maintaining secure backups.

Finding Potential Security Risks

Several free online tools can help identify suspicious websites, malicious files, and unusual network activity.

Tool Purpose
VirusTotal Scans files and URLs using dozens of antivirus engines.
URL Reputation Services Checks whether websites are considered safe before visiting them.
Network Monitoring Helps identify applications making unexpected network connections.
Security Tip
  • Verify website addresses before logging in.
  • Scan unknown downloads before opening them.
  • Monitor unusual network activity.
  • Keep browsers and antivirus software updated.
  • Never ignore browser security warnings.

Useful Resources:

  • VirusTotal
  • Google Safe Browsing

Types of Security Controls

Organizations use several layers of security controls to reduce cyber risk. No single control can stop every attack, so companies combine multiple approaches to create a strong defense.

Control Type Examples Purpose
Technical Firewalls, Antivirus, Encryption, MFA Protect systems using technology.
Managerial Security Policies, Risk Assessments, Audits Guide security planning and governance.
Operational Security Awareness Training, Incident Response Improve daily security procedures.
Physical Badge Readers, Cameras, Locks, Biometrics Protect facilities and equipment.
Defense in Depth

Good cybersecurity relies on multiple layers of protection rather than a single security solution.

Cryptography and Public Key Infrastructure (PKI)

Encryption protects sensitive information by converting readable data into an unreadable format that only authorized users can access.

One of the most widely used cryptographic systems is Public Key Infrastructure (PKI), which uses two mathematically related keys.

Key Purpose
Public Key Encrypts data and verifies digital signatures. It can be shared freely.
Private Key Decrypts encrypted data and creates digital signatures. It must remain secret.
Common PKI Technologies
  • Digital Certificates
  • Certificate Authorities (CA)
  • Hardware Security Modules (HSM)
  • Trusted Platform Modules (TPM)

PKI helps secure websites, email communication, VPN connections, software updates, and many other everyday technologies.

Change Management

Every modification to software, hardware, or network infrastructure should follow a formal change management process.

Proper planning reduces downtime, prevents configuration mistakes, and minimizes security risks caused by unauthorized or poorly tested changes.

Step Description
Request Document the proposed change.
Review Assess security and business impact.
Approval Receive management authorization.
Implementation Deploy the approved change.
Verification Confirm the change was successful.
Documentation Record the completed work.
Why It Matters

Structured change management improves security, accountability, system stability, and compliance.
